Project Overview

This project focuses on building and deploying a multilingual AI research assistant that retrieves domain-specific insights directly from academic papers using a vector database and agent-based orchestration. The system allows users to upload research papers, ask questions in any language, and receive context-aware answers grounded in indexed academic sources.

The Challenge

Modern large language models are powerful, but they face two major limitations in research settings:

  1. Hallucination Risk – LLMs can generate plausible but incorrect academic information when not grounded in verified sources.
  2. Domain Isolation – Researchers often need answers strictly based on a curated set of papers, not general internet knowledge.
  3. Multilingual Accessibility – Academic tools are often limited to English, creating barriers for international collaboration.
  4. Scalable Deployment – Research tools must handle document indexing, embeddings, retrieval, translation, and inference across distributed systems without breaking.

The challenge was to design a system that could:

  • Retrieve accurate, document-grounded answers from uploaded academic papers
  • Support multilingual queries without corrupting semantic meaning
  • Transparently show which research papers were used
  • Prevent hallucinations through controlled workflow orchestration
  • Deploy reliably in a cloud-native, containerized infrastructure

The Solution

Agent-Orchestrated Architecture - The system uses an LLM as a central orchestration node, built with LangGraph, to manage:

  • Query understanding
  • Translation routing
  • Vector retrieval from Milvus
  • Response synthesis grounded in retrieved documents

Vector Database for Context Retrieval - A Milvus vector database stores embeddings for academic papers. When a query is submitted:

  • The question is embedded
  • Milvus performs semantic similarity search
  • The top relevant documents are retrieved
  • The LLM generates a response grounded strictly in that context
  • The top two relevant papers are displayed alongside the answer

Multilingual Translation Service - To enable Multilingual queries, I deployed a containerized translation service on GKE that utilized MCP to communicate with the agent.

Cloud-Native Deployment on GKE - The entire system was deployed on Google Kubernetes Engine (GKE) using containerized services:

  • AI Agent service
  • MCP translation service
  • Streamlit front-end
  • Milvus vector database cluster
