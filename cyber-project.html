Cyber Attack Detection Predictor (MQTT)

PySpark SQL PyTorch Kafka GCP

Project Overview

This project focuses on designing and deploying a scalable data analytics and machine learning pipeline for cybersecurity traffic analysis using real-world MQTT network data.

The Challenge

The widespread adoption of IoT systems has made lightweight protocols like MQTT essential for efficient device communication. However, MQTTâ€™s simplicity also exposes IoT networks to security threats such as denial-of-service attacks, unauthorized access, and message flooding. Identifying these attacks is difficult due to the scale and variability of IoT traffic and the subtle nature of malicious behavior. Addressing these challenges requires scalable analytics and machine learning techniques capable of detecting attacks in real-world MQTT network data.

The Solution

I designed an end-to-end data pipeline that emphasized scalability, modularity, and reproducibility::

  • Data Ingestion & Storage: Both training and testing MQTT datasets were ingested into a PostgreSQL database under a dedicated schema, with explicit labeling to distinguish dataset splits.
  • Distributed Analytics with Spark: All core analytics were implemented using PySpark, enabling efficient processing of large datasets directly from the database.
  • Publisher/Subscriber Architecture: Kafka was used to ingest and analyze live YouTube comments related to cybersecurity topics, mapping real-world discussion trends to known attack categories.
  • Machine Learning Modeling: Multiple classifiers were built and evaluated in both Spark ML and PyTorch, with systematic hyperparameter tuning and performance comparison to identify the most effective models.
  • Cloud Deployment: The entire workflow was adapted to run in a cloud environment, ensuring the project could scale beyond local hardware limitations and mirror real-world production constraints.
