Currently working on a project that focuses on productionizing the semantic-spatial robotic framework for real-world deployments with natural language-based interaction.

The Challenge

Today, many robotics projects that incorporate natural language understanding rely heavily on cloud-based API calls to large language models. While these models are powerful, this approach introduces several limitations for real-time robotic systems. Network latency and dependency on a stable internet connection can hinder responsiveness, which is critical for interactive or safety-sensitive applications. Additionally, frequent API usage can become costly over time, making large-scale deployment or long-term experimentation impractical for student projects, research labs, or low-budget systems. These constraints motivate the exploration of locally deployable language models and alternative architectures that better support autonomous robotic behavior.

The Solution

Modernize a framework that allows for direct language comparisons and distill larger models to allow for efficient local deployment.
